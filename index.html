로컬 서버

학교 컴퓨터실 서버 PC에서 직접 실행. 같은 네트워크 내에서만 접속 가능.

1

Supabase 프로젝트 생성

데이터베이스와 인증을 담당합니다

  1. supabase.com에 가입/로그인합니다
  2. New Project 클릭
  3. 설정 입력:
    • Name: pythink2 (원하는 이름)
    • Database Password: 안전한 비밀번호 (메모!)
    • Region: Northeast Asia (Seoul)
  4. 생성 완료 후 Settings > API에서 아래 값을 메모합니다:
    Project URL→SUPABASE_URL
    anon public 키→SUPABASE_ANON_KEY
    service_role 키→SUPABASE_SERVICE_ROLE_KEY
⚠️
service_role 키는 절대 외부에 노출하지 마세요. 이 키로 DB의 모든 데이터를 읽고 쓸 수 있습니다.
2

데이터베이스 테이블 생성

SQL 파일을 Supabase에서 실행합니다

  1. Supabase 대시보드에서 SQL Editor 클릭
  2. GitHub의 supabase/migrations/ 폴더에서 SQL 파일들을 번호 순서대로 실행합니다
# 실행 순서 (번호가 작은 것부터)
20260226000000_initial_schema.sql       ← 가장 먼저
20260226100000_hrs_schema.sql
20260226200000_learning_events.sql
20260226300000_indexes_rls.sql
20260226400000_achievement_system.sql
... (나머지도 순서대로)
💡
각 파일을 열어 내용을 복사 → SQL Editor에 붙여넣기 → Run 버튼. 또는 supabase/schema.sql로 한 번에 실행할 수도 있습니다.
3

Supabase 인증 설정

이메일 인증을 활성화합니다

  1. Authentication > Providers에서 Email 활성화 (기본 활성)
  2. Settings > Authentication에서:
    • Site URL: 나중에 Vercel URL로 변경
    • Redirect URLs에 추가: https://내학교.vercel.app/**

  2. API Keys 메뉴에서 새 키 생성
  3. sk-ant-...로 시작하는 키를 안전하게 메모

Railway에 AI 서버 배포

AI 코칭과 분류 엔진을 담당하는 서버입니다

  1. railway.app에 GitHub 계정으로 로그인
  2. New Project > Deploy from GitHub repo > Fork한 저장소 선택
  3. Settings에서 Start Command: node server/index.js
  4. Variables 탭에서 환경 변수 추가 (아래 표 참고)
  5. Deploy 클릭!
변수명값
PORT4004
CLIENT_URLVercel URL (7단계 후 수정)
SUPABASE_URL1단계에서 메모한 URL
SUPABASE_ANON_KEY1단계에서 메모한 anon 키
SUPABASE_SERVICE_ROLE_KEY1단계에서 메모한 service_role 키
ANTHROPIC_API_KEY4단계에서 메모한 sk-ant-... 키
ADMIN_API_KEY직접 만든 관리자 키 (16자 이상)
API_KEY_ENCRYPTION_SECRET아래 명령으로 생성
DEMO_PASSWORDdemo-pythink2-2026
🔑
API_KEY_ENCRYPTION_SECRET 생성법 (터미널에서): 
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
64자리 16진수 문자열이 출력됩니다. 복사해서 붙여넣으세요.

배포 완료 후 제공된 URL을 메모하세요. 브라우저에서 https://배포URL/api/health에 접속하여 {"status":"ok"}이 보이면 성공!

Vercel에 프론트엔드 배포

학생/교사가 접속하는 웹사이트입니다

  1. vercel.com에 GitHub 계정으로 로그인
  2. Add New > Project > Fork한 저장소 선택
  3. 설정:
    • Framework Preset: Vite
    • Build Command: cd client && npm install && npm run build
    • Output Directory: client/dist
  4. Environment Variables 추가:
    변수명값
    VITE_SUPABASE_URLSupabase URL
    VITE_SUPABASE_ANON_KEYSupabase anon 키
    VITE_API_URLRailway URL (6단계)
    VITE_DEMO_PASSWORDdemo-pythink2-2026
    VITE_DEMO_ACCESS_CODE1234
  5. Deploy 클릭!

URL 상호 연결 (중요!)

세 서비스가 서로를 알 수 있도록 연결합니다

Railway 수정:

CLIENT_URL을 Vercel URL로 업데이트

Supabase 수정:

Authentication > URL Configuration에서 Site URL과 Redirect URLs 설정

vercel.json 수정:

Fork한 GitHub에서 vercel.json의 Railway URL을 내 것으로 변경:

{
  "rewrites": [
    {
      "source": "/api/(.*)",
      "destination": "https://내-Railway-URL.up.railway.app/api/$1"
    },
    { "source": "/(.*)", "destination": "/index.html" }
  ]
}

다른 기기에서 접속하기

같은 학교 Wi-Fi 네트워크 안에서 가능합니다

  1. 서버 PC의 IP 주소 확인 (예: 192.168.1.100)
  2. 다른 기기(태블릿/노트북)에서 http://192.168.1.100:4003 접속

⚠️
로컬 서버 방식에서도 인터넷 연결은 필수입니다 (Supabase DB + Anthropic AI API 사용). 방화벽에서 4003, 4004 포트를 열어주세요.

이미 적용된 보안 조치

pythink2에 내장된 보호 장치들입니다

영역보호 조치상세
데이터 격리 RLS (Row Level Security) 전체 18개 테이블에 행 수준 보안 적용. 학생은 자기 데이터만, 교사는 자기 교실만 접근 가능
인증 JWT + Supabase Auth 모든 API 요청에 토큰 검증. 30분 비활동 시 자동 로그아웃
권한 분리 역할 기반 접근 제어 학생/교사/관리자 3단계. 역할 변경 불가(DB 레벨 잠금)
통신 암호화 HTTPS 전용 Vercel/Railway 모두 HTTPS 강제. 보안 헤더(Helmet.js) 적용
API 보호 Rate Limiting + 큐 학생당 분당 20회, 동시 AI 처리 3건 제한. DDoS 방어
API 키 암호화 AES-256-GCM 교실별 API 키를 DB에 암호화 저장. 평문 노출 없음
AI 안전 출력 필터링 AI 코치 응답에서 실행 가능 코드(정답) 자동 차단
CORS 화이트리스트 허용된 도메인에서만 API 접근 가능
개인정보처리방침 10개 조항 앱 내 개인정보처리방침 페이지 탑재 (개인정보보호법 준수)

학습 기록은 학년 종료 후 최대 1년 보유 후 파기됩니다. 교사가 교실을 삭제하면 해당 교실의 모든 데이터가 즉시 삭제(CASCADE)됩니다.

학교 간 데이터 완전 분리

다른 학교 선생님과 데이터가 섞이지 않습니다

A 학교 선생님
자체 Supabase
A 학교 학생 데이터만
B 학교 선생님
자체 Supabase
B 학교 학생 데이터만
C 학교 선생님
자체 Supabase
C 학교 학생 데이터만

각 선생님이 별도 Supabase 프로젝트를 생성하므로, 데이터베이스 자체가 물리적으로 분리됩니다.
다른 학교의 데이터에 접근하는 것은 기술적으로 불가능합니다.

!

선생님이 반드시 지켜야 할 보안 수칙

시스템 보안이 아무리 좋아도 이것들을 지키지 않으면 무용지물입니다

1. 환경 변수(키)를 절대 공유하지 마세요

SUPABASE_SERVICE_ROLE_KEY, ANTHROPIC_API_KEY 등은 카카오톡, 이메일, 문서에 절대 적지 마세요.
특히 service_role 키가 유출되면 DB의 모든 학생 데이터에 접근 가능합니다.

2. CSV 내보내기 파일을 안전하게 관리하세요

교사 대시보드에서 내보낸 CSV에는 학생 이메일과 학습 기록이 포함됩니다.
USB, 클라우드, 공유 폴더에 방치하지 마세요. 사용 후 삭제를 권장합니다.

3. AI 코칭 데이터 전송에 대해 안내하세요

학생이 AI 코치를 사용하면 학습 데이터(코드, 질문)가 Anthropic(미국)으로 전송됩니다.
개인 식별 정보(이름, 이메일)는 전송되지 않지만, 학기 초에 학생/학부모에게 안내하는 것을 권장합니다.

4. 학생 이메일은 닉네임 이메일을 권장합니다

실명 이메일 대신 3반7번@학교메일 같은 닉네임 이메일을 사용하면 개인 식별 위험이 크게 줄어듭니다.
또는 학교 구글 워크스페이스 계정을 활용하세요.

5. 학기 종료 후 교실을 삭제하세요

교실 삭제 시 해당 교실의 모든 학생 데이터(제출, 프로필, 학습 기록)가 자동으로 완전 삭제됩니다.
필요한 통계는 CSV로 먼저 내보낸 후 교실을 삭제하세요.

6. Anthropic API 사용량 한도를 설정하세요

console.anthropic.com에서 월 사용 한도를 설정하면 예상치 못한 비용 발생을 방지할 수 있습니다.
교실 설정의 daily_ai_coach_limit으로 학생당 일일 사용을 제한할 수도 있습니다.

키가 유출된 것 같다면?

즉시 조치 순서

  1. Anthropic API 키 유출: console.anthropic.com에서 즉시 키 삭제 → 새 키 발급 → Railway 환경 변수 교체
  2. Supabase service_role 키 유출: Supabase 대시보드 > Settings > API에서 키 재생성 → Railway + 시딩 스크립트의 키 모두 교체
  3. ADMIN_API_KEY 유출: Railway 환경 변수에서 새 키로 교체 (즉시 적용)
  4. 의심스러운 활동이 있었다면 Supabase > Authentication > Users에서 비정상 계정 확인

자주 묻는 질문 (FAQ)

사이트에 접속했는데 빈 화면이 나와요
브라우저 개발자 도구(F12) > Console 탭을 확인하세요.
  • SUPABASE_URL 에러 → 클라이언트 환경 변수 확인
  • CORS 에러 → Railway의 CLIENT_URL이 Vercel URL과 일치하는지 확인
AI 코치가 응답하지 않아요
  1. Railway 서버 실행 확인: https://서버URL/api/health
  2. ANTHROPIC_API_KEY가 유효한지 확인
  3. Railway 로그에서 에러 메시지 확인
학생이 회원가입이 안 돼요
  • Supabase > Authentication > Providers에서 Email 활성화 확인
  • Supabase > URL Configuration에서 Site URL 확인
문제 목록이 비어있어요
시딩 스크립트를 실행했는지 확인하세요: 
node scripts/seed-all.js
로컬에서 다른 컴퓨터로 접속이 안 돼요
  • 서버 PC 방화벽에서 4003, 4004 포트 허용
  • CLIENT_URL에 서버 PC의 IP 주소 추가
  • 같은 Wi-Fi/네트워크에 있는지 확인
Supabase 무료 플랜 한계가 있나요?
Free 플랜 기준:
  • 데이터베이스 500MB (학생 200명 x 1년 충분)
  • 인증 사용자 50,000명
  • API 요청 무제한
  • 7일 비활성 시 일시 정지 (접속하면 자동 재시작)
Anthropic API 비용이 걱정돼요
Anthropic API는 필수가 아닙니다. API 키 없이도 문제 풀이, 교사 대시보드, 추천, 로드맵 등 모든 기본 기능이 정상 작동합니다. AI 코칭(힌트/피드백 채팅)만 비활성화됩니다.
  • AI 코칭을 사용할 경우 학생 30명 기준 월 $5~15 예상
  • console.anthropic.com에서 사용량 한도 설정 가능
  • 교실 설정의 daily_ai_coach_limit으로 학생당 일일 사용 제한
"Rate limit exceeded" 에러가 나와요
AI 요청이 너무 많을 때 발생합니다. 기본 제한:
  • 학생당 분당 20회 AI 요청
  • 동시 AI 처리 3건
교실 설정에서 일일 AI 코치 제한을 조절하세요.
